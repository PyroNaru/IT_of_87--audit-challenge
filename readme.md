Frontend Code Audit & Refactor Challenge

Project Description:
This project is part of the Frontend Code Audit & Refactor Challenge.  
The goal of the activity is to analyze a broken frontend project, identify issues in HTML, CSS, file structure, and Git usage, then refactor the code.

Objectives:

- Audit existing frontend code
- Identify HTML, CSS, and structural issues
- Document findings in `AUDIT.md`
- Refactor the project using semantic HTML and modern CSS
- Practice proper GitHub workflow and commits
- Improve project structure and responsiveness

Features:

- Semantic HTML structure (`header`, `main`, `section`, `footer`)
- Improved and responsive layout using Flexbox/Grid
- Cleaned and organized file structure
- Fixed broken links and images
- Free, locally stored images (no hotlinking)
- Clear documentation and audit report
